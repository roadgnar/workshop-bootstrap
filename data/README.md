# Cyvl's Public Data Repository

Cyvl has shared publicly datasets from a number of projects across the United States in an effort to support research and public good initiatives. These datasets include rich GIS layers describing the location and condition of various infrastructure assets such as Traffic Signs, Pavement Striping & Markings (lane paint), the condition of the road surface itself, detected pavement distresses, and rich geolocated Imagery and LiDAR pointclouds captured at the street level.

**Legend:**
- **Pavement Data**: Includes 30-foot PCI scores (pavements.geojson), centerline rollup scores (rollup.geojson), and/or pavement distress detections (distresses.geojson)
- **Asset Inventory**: Above-ground assets such as sidewalks, signs, traffic signals, utilities, etc.
- **Imagery**: Street-level imagery (streetviewImages.geojson) and/or plain imagery (plainImagery.geojson)

For detailed information about each dataset, see the project-specific README files in each project directory.

## Typical project structure

### Shared Project Structure

- `project/`
  - `pointclouds/` - .LAZ Pointcloud data, geolocated in UTM 
  - `gis/`
    - `pavement/` — Pavement condition GIS layers (PCI/PASER scores)  
    - `assets/` — Above-ground-Assets/Right-of-Way (ROW) GIS features  
    - `imagery/` — Image layer GIS files  
  - `README.md` — Project specific Documentation

## Cyvl Data Schemas

This section documents the schema for each GeoJSON file type included in shared datasets. All files use standard GeoJSON format with geometry and properties.

### Pavement Data

#### `pavements.geojson` - 30 foot PCI pavement scores
**Geometry Type:** LineString  
**Description:** Pavement condition assessment at approximately 30-foot intervals along road segments.

**Properties:**
- `inspect_id` (string): Unique inspection identifier for this pavement segment
- `client_seg_id` (string): Client segment identifier linking to road network
- `address_st` (string): Street name or address where the segment is located
- `score` (float): PCI (Pavement Condition Index) score. Values typically range from 0-100, where higher values indicate better condition. May be null for unscored segments.
- `label` (string): Categorical condition label. Common values: "Very Poor", "Poor", "Fair", "Satisfactory", "Good", "Serious", "Failed", "Not Scored"
- `stroke` (string): Hex color code used for visualization (corresponds to label/condition)
- `length_ft` (integer): Length of the pavement segment in feet
- `area_sqft` (integer): Area of the pavement segment in square feet
- `lat` (float): Latitude coordinate (WGS84)
- `lon` (float): Longitude coordinate (WGS84)

#### `rollup.geojson` - Centerline-level PCI pavement scores
**Geometry Type:** LineString  
**Description:** Aggregated pavement condition scores at the centerline/street segment level.

**Properties:**
- `client_seg` (string): Client segment identifier (matches client_seg_id in pavements.geojson)
- `score` (float): Aggregated PCI score for the entire segment (0-100 scale)
- `label` (string): Categorical condition label (same values as pavements.geojson)
- `stroke` (string): Hex color code for visualization

#### `distresses.geojson` - Individual pavement distress detections
**Geometry Type:** Polygon  
**Description:** Individual distress features detected on pavement surfaces.

**Properties:**
- `id` (string): Unique identifier for the distress feature
- `cell_distress_id` (integer): Identifier linking to inspection cell
- `dis_un_id` (string): Distress union identifier (groups related distresses)
- `client_seg_id` (string): Client segment identifier
- `inspect_id` (string): Inspection identifier (e.g., "IA00034")
- `distress_type` (string): Type of distress detected. Common values: "potholes", "raveling", "bleeding_flushing", "lt_cracking", "depressions", "rutting", "sealant", "block_cracking", "weathering", "edge_cracking", "alligator_cracking", "longitudinal_cracking", "transverse_cracking", "patches"
- `severity` (string): Severity level. Values: "low", "medium", "high"
- `area_sq_ft` (float): Area of the distress in square feet
- `length_ft` (float): Length of the distress in feet (for linear distresses)
- `image` (string): URL to image showing the distress

### Asset Data

#### `aboveGroundAssets.geojson` - Above-ground asset inventory
**Geometry Type:** Point or LineString (varies by asset type)  
**Description:** Inventory of above-ground assets and right-of-way features detected along roadways.

**Properties:**
- `feature_id` (string): Unique identifier for the asset feature
- `asset_type` (string): Type of asset. Common values include: "SIDEWALK", "MANHOLE_COVER", "HYDRANT", "TRAFFIC_SIGNAL", "UTILITY_POLE", "CATCH_BASIN", "SIGNAL_POLE", "LUMINARIES", "TREE", "BENCH", "BIKE_RACK", "TRASH_BIN", "GUARDRAILS", "CURB", "RAMP", "CURB_CUT", "DRIVEWAY", "RETAINING_WALL", "JERSEY_BARRIER", "CCTV", "FLASHER", "PEDESTRIAN_PUSH_BUTTON", "PLANTER", "SIDEWALK_OBSTRUCTION", "VALVE_COVER", "CABINET", "BOX", "BUS_SHELTER", "BUS_STOP", "FENCE", "BANNER", "PARKING_WITHIN_20FT_OF_INTERSECTION", "STAND_ALONE_PED_PUSH_BUTTON", "STAND_ALONE_PEDESTRIAN_HEAD", "TRAFFIC_SIGNAL_POLE", "WATER_VALVE"
- `image_url` (string): URL to image showing the asset
- `condition` (string, optional): Condition assessment. Values: "Good", "Fair", "Poor", "Under Construction", "Cant Verify", "None"
- `Material` (string, optional): Material type (e.g., "Concrete", "Asphalt", "Brick", "Gravel", "Other", "Cant Verify", "None")
- `Type` (string, optional): Subtype classification (e.g., for sidewalks: "Sidewalk", "Bike_path", "No Sidewalk")
- `Condition` (string, optional): Alternative condition field (may be present in some datasets)

#### `sam.geojson` - Striping and Markings inventory
**Geometry Type:** LineString  
**Description:** Pavement striping and road markings including lane lines, crosswalks, arrows, and other painted features.

**Properties:**
- `feature_id` (string): Unique identifier for the marking feature
- `category` (string): Category of marking. Values: "striping", "markings"
- `type` (string): Specific type of marking. Examples: "STOP", "BIKE LANE GREEN STRIPED", "SPEED CUSHION", "STRAIGHT ARROW", "STANDARD CROSSWALK", "RIGHT TURN ARROW", "LADDER CROSSWALK", "BIKE SYMBOL", "LANE", "AHEAD OR RIGHT TURN ARROW", "LEFT TURN ARROW", "YIELD", "DO NOT ENTER", "ONE WAY", "PARKING", "NO PARKING", "FIRE LANE", "HANDICAP", "LOADING ZONE", "BUS STOP", "SCHOOL ZONE", "SPEED LIMIT", "STOP AHEAD", "YIELD AHEAD", "SCHOOL CROSSING", "PEDESTRIAN CROSSING", "RAILROAD CROSSING", "DEER CROSSING", "BIKE CROSSING", "HOSPITAL", "HOTEL", "GAS", "FOOD", "LODGING", "REST AREA", "SCENIC VIEW", "PICNIC AREA", "CAMPING", "TRAIL", "OTHER"
- `line_type` (string, optional): Type of line. Values: "SINGLE", "DOUBLE", "SOLID", "DASHED", "CROSSWALK", "BIKE LANE", "YIELD", "STOP BAR", "OTHER"
- `line_subtype` (string, optional): Subtype of line. Values: "STANDARD", "WIDE STRIPE", "WIDE DASHED", "DASHED", "ZEBRA", "CONTINENTAL", "LADDER", "STOP BAR", "OTHER"
- `color` (string, optional): Color of the marking. Values: "White", "Yellow", "Green", "Brick Or Other", "Other"
- `condition` (string): Condition assessment. Values: "Good", "Fair", "Poor", "Under Construction", "Cant Verify", "None"
- `comment` (string, optional): Additional comments or notes
- `image_url` (string): URL to image showing the marking
- `location` (null/string, optional): Location description (typically null)

#### `signs.geojson` - Traffic signs inventory
**Geometry Type:** Point  
**Description:** Inventory of traffic signs detected along roadways.

**Properties:**
- `fid` (integer): Feature identifier
- `FEAT_ID` (float): Feature ID (may duplicate fid)
- `category` (string): Sign category. Values: "Regulatory", "Warning", "Guide", "School", "Marker", "Other", "Other-mutcd"
- `mutcd` (string): MUTCD (Manual on Uniform Traffic Control Devices) code (e.g., "R3-5R", "W3-3", "R6-1L", "W17-1", "R3-3", "R10-3", "R3-8", "other", "other_bus_stop")
- `MUTCD` (string, optional): Alternative MUTCD field (may be empty)
- `condition` (string): Condition assessment. Values: "Good", "Fair", "Poor"
- `image_url` (string): URL to annotated image showing the sign
- `image_basename` (string): Basename of the annotated image file
- `basename` (string): Basename of the source image
- `location` (string): JSON string containing coordinates: `{"lat": <latitude>, "lon": <longitude>}`
- `layer` (string): Annotation layer/task identifier (e.g., "annotations_task_3459")
- `path` (string): File path to source annotation file
- `updated` (string): Update status (typically "Yes")
- `uuid` (string): UUID identifier (may be empty)
- `obj_range` (integer): Object range (typically 0)
- `comment` (null): Comment field (typically null)
- `image_point` (null): Image point field (typically null)

### Imagery Data

#### `streetviewImages.geojson` - Street-level imagery points
**Geometry Type:** Point  
**Description:** Georeferenced points for street-level imagery captured along roadways.

**Properties:**
- `id` (string): Unique image identifier (e.g., "GX011077_MP4_1753198084787476_front_3000")
- `image_url` (string): URL to the street-level image
- `lat` (float): Latitude coordinate of image capture location (WGS84)
- `lon` (float): Longitude coordinate of image capture location (WGS84)
- `bearing` (float): Camera bearing/direction in degrees (0-360)
- `timestamp` (float): Timestamp of image capture (Unix timestamp in microseconds)
- `next_id` (string, optional): ID of next image in sequence (may be null for last image)
- `prev_id` (string): ID of previous image in sequence

#### `streetviewImagePaths.geojson` - Street-level imagery path network
**Geometry Type:** Point  
**Description:** Network representation of street-level imagery paths, linking consecutive images.

**Properties:**
- Same as `streetviewImages.geojson` (id, image_url, lat, lon, bearing, timestamp, next_id, prev_id)

#### `plainImagery.geojson` - Plain street-level imagery points
**Geometry Type:** Point  
**Description:** Georeferenced points for plain street-level imagery (alternative format to streetviewImages).

**Properties:**
- Same as `streetviewImages.geojson` (id, image_url, lat, lon, bearing, timestamp, next_id, prev_id)

#### `panoramicImagery.geojson` - 360-degree panoramic imagery points
**Geometry Type:** Point  
**Description:** Georeferenced points for 360-degree panoramic imagery captured along roadways. Unlike traditional forward-facing street-level imagery, these images provide a full 360-degree view from each capture location.

**Properties:**
- `id` (string): Unique image identifier (e.g., "GS010002_360_1756924813656817_360_3000")
- `image_url` (string): URL to the 360-degree panoramic image viewer (typically points to platform.cyvl.ai/3DViewer.html with the image URL as a parameter)
- `lat` (float): Latitude coordinate of image capture location (WGS84)
- `lon` (float): Longitude coordinate of image capture location (WGS84)
- `bearing` (float): Camera bearing/direction in degrees (0-360)
- `timestamp` (float): Timestamp of image capture (Unix timestamp in microseconds)
- `next_id` (string): ID of next image in sequence
- `prev_id` (string): ID of previous image in sequence

### Pointcloud Data

#### `pointcloud_coverage.json` - LiDAR pointcloud coverage and download links
**File Format:** GeoJSON FeatureCollection  
**Geometry Type:** Polygon  
**Description:** Coverage polygons showing where high-resolution LiDAR pointclouds are available. Each feature represents a pointcloud tile with public download links to the corresponding .LAZ files.

**Properties:**
- `filename` (string): Name of the .LAZ pointcloud file (e.g., "global_xyz_rgb_icgu_0_0_2000.laz")
- `filename_no_ext` (string): Filename without the .laz extension
- `baseUrl` (string): Base URL for cloud storage (e.g., "https://dcygqrjfsypox.cloudfront.net")
- `lasPath` (string): Relative path to the .LAZ file. Combine with `baseUrl` to create the full download URL: `{baseUrl}{lasPath}`
- `potreePath` (string): Relative path to the Potree visualization format. Combine with `baseUrl` to create the full URL: `{baseUrl}{potreePath}`
- `dataset` (string): Dataset identifier (e.g., "scan_1049_BEVERLY_MA")
- `dataset_encoded` (string): Encoded dataset identifier used in file paths
- `lon` (float): Longitude coordinate of the tile center point (WGS84)
- `lat` (float): Latitude coordinate of the tile center point (WGS84)
- `alt` (float): Altitude/elevation of the tile center point in meters

**Note:** The polygon geometry represents the spatial extent/coverage area of each pointcloud tile. To download a .LAZ file, construct the URL as: `{baseUrl}{lasPath}`.

#### `.LAZ` Pointcloud File Format

**File Format:** LAZ (compressed LAS)  
**LAS Version:** 1.4  
**Point Format:** 7  
**Description:** High-resolution LiDAR pointcloud data stored in compressed LAZ format. Pointclouds are **geolocated to their UTM grid** and the **altitude (Z coordinate) is specified in WGS84 Height Above Ellipsoid**.

**Point Data Fields:**

| Field | Type | Description |
|-------|------|-------------|
| X, Y, Z | double | 3D coordinates. X and Y are in UTM meters, Z is WGS84 Height Above Ellipsoid in meters |
| Intensity | unsigned short | Return intensity value (0-65535) |
| Return Number | unsigned char | Return number for this pulse (1-15) |
| Number of Returns | unsigned char | Total number of returns for this pulse (1-15) |
| Scanner Channel | unsigned char | Scanner channel number |
| Scan Direction Flag | bool | Direction of scan (0 = forward, 1 = backward) |
| Edge of Flight Line | bool | Edge of flight line flag (0 = not edge, 1 = edge) |
| Classification | unsigned char | Point classification (ASPRS standard) - **Coming soon, not yet populated** |
| Synthetic Flag | bool | Synthetically generated point (0 = real, 1 = synthetic) |
| Keypoint Flag | bool | Keypoint flag (0 = not keypoint, 1 = keypoint) |
| Withheld Flag | bool | Withheld flag (0 = not withheld, 1 = withheld) |
| Overlap Flag | bool | Overlap flag (0 = not overlap, 1 = overlap) |
| Scan Angle | char | Scan angle rank (-90 to +90 degrees) |
| User Data | unsigned char | User-defined data field |
| Point Source ID | unsigned short | Point source ID |
| GPS Time | double | GPS time of point acquisition |
| Red | unsigned short | Red color channel (0-65535) - **Currently unused and unpopulated** |
| Green | unsigned short | Green color channel (0-65535) - **Currently unused and unpopulated** |
| Blue | unsigned short | Blue color channel (0-65535) - **Currently unused and unpopulated** |
| Near Infrared | unsigned short | Near infrared channel (0-65535), if available |

**Important Notes:**
- **Coordinate System:** X and Y coordinates are in UTM (Universal Transverse Mercator) meters. The specific UTM zone varies by project location.
- **Altitude:** Z coordinate (altitude) is in **WGS84 Height Above Ellipsoid** in meters, not orthometric height (MSL).
- **Compression:** Files are compressed using LAZ format for efficient storage and transfer.
- **Point Count:** Individual tile files may contain millions to hundreds of millions of points.

---

**Note:** Some properties may vary between projects. Optional fields may be null or absent in certain datasets. Image URLs point to cloud storage and may require authentication or have expiration dates depending on the sharing arrangement.
